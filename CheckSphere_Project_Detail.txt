CHECKSPHERE - COMPLETE PROJECT BLUEPRINT (A TO Z)
==================================================

1. PROJECT OVERVIEW
-------------------
CheckSphere is a next-generation, full-stack attendance intelligence system. 
It automates classroom attendance while eliminating proxy attendance (where 
students mark attendance for absent friends) through high-tech verification.

2. CORE TECHNOLOGIES (THE STACK)
--------------------------------
• Frontend: Next.js 15 (App Router), React 19
• Styling: Tailwind CSS v4, Framer Motion (Animations)
• Backend: Next.js API Routes (Serverless)
• Database: PostgreSQL (Cloud-hosted on Neon.tech)
• ORM: Prisma (Type-safe SQL access)
• Auth: JSON Web Tokens (JWT) & Bcrypt Hashing

3. DATABASE ARCHITECTURE (RELATIONAL SQL)
-----------------------------------------
The system uses a relational model to ensure data integrity:
• User Table: Stores roles (Admin, Teacher, Student), hashed passwords, and profile data.
• Session Table: Stores live class data including Teacher ID, GPS Coordinates, and Dynamic Secrets.
• Attendance Table: Records the link between a Student, a Session, and their unique Device Fingerprint.

4. KEY TECHNICAL FEATURES
-------------------------
A. INTELLIGENT GEOFENCING:
   Uses the Haversine Formula to calculate the distance between the student's phone 
   and the classroom. Check-ins are rejected if the student is outside the 50m radius.

B. ANTI-PROXY INTELLIGENCE:
   • Device Fingerprinting: Captures unique browser/hardware IDs to block multiple accounts on one phone.
   • Dynamic Session Codes: Rotating 6-digit codes generated by teachers for real-time verification.

C. ROLE-BASED DASHBOARDS (RBAC):
   Custom interfaces for different users:
   • Teachers: Session management, live tracking, and exportable reports.
   • Students: Contactless "Terminal" interface for GPS-verified check-ins.
   • Admins: Global system analytics and user management.

5. SECURITY & PERFORMANCE
-------------------------
• Stateless Auth: Secure JWT tokens handle all sessions without overloading the server.
• Password Hashing: Uses 10-round Bcrypt salt for military-grade data protection.
• Global Edge: Deployed on Vercel's global network for sub-100ms response times.

6. CLOUD INFRASTRUCTURE
-----------------------
• Hosting: Vercel (Production Environment)
• Database: Neon (Serverless PostgreSQL)
• CI/CD: Automated deployment via GitHub (https://github.com/Saketh-Mattaparthi/CheckSphere)

7. DESIGN SYSTEM (ANTIGRAVITY UI)
---------------------------------
CheckSphere uses a custom "AntiGravity" glassmorphic design system:
• Frosted-glass panels with backdrop-blur effects.
• Dynamic color gradients (Primary: Indigo/Violet, Accent: Cyan/Emerald).
• Micro-animations for feedback on every user interaction.
